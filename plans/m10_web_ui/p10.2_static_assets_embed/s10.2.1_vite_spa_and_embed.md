# S10.2.1 — Vite SPA + production embed

**Phase:** P10.2 (static assets + production embed)  
**Estimate:** 2-3 weeks  
**Status:** stage placeholder — to be expanded when picked up.

## Context

Frontend build pipeline + the trick that makes the binary
self-contained in release.

## Likely tasks

### T10.2.1.1 — Vite + framework choice

Stand up the SPA shell. Recommended: **Preact + Vite** (TypeScript
strict mode, JSX). Alternatives evaluated up-front: Solid (if
fine-grained reactivity matters more), vanilla TS + lit-html
(smallest bundle, more work). React deliberately not picked —
unnecessary heft for a single-window operator UI.

`web/` directory at repo root. Vite config: dev server on
:5173 with HMR; build output to `web/dist/`.

### T10.2.1.2 — Disk-served dev mode

`acva` in dev mode serves static assets from `web/dist/`
(read at runtime, refresh-on-change for the operator's sanity).
Gated on `cfg.web_ui.dev_mode: true`.

### T10.2.1.3 — Embedded release mode

Release builds embed `web/dist/*` into the `acva` binary via
generated C++ resources (`xxd -i` style or `cmrc`
(`CMakeRC`)). Static handler reads from the in-memory bundle
on `GET /` and `GET /assets/*`. Compression: gzip the embedded
assets, `Content-Encoding: gzip` to the browser.

### T10.2.1.4 — CMake plumbing

`web/dist/` builds via `npm ci && npm run build` triggered
by CMake. Skip silently if `npm` is missing (release builds
work even without Node when assets are pre-built and cached).

### T10.2.1.5 — Acceptance

`./build.sh release && ./_build/release/acva` and opening
`http://127.0.0.1:9876/` shows the SPA shell. No external
network dependencies.
