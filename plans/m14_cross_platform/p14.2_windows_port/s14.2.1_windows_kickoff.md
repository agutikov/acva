# S14.2.1 — Windows port kickoff

**Phase:** P14.2 (Windows port)
**Estimate:** 4-5 weeks
**Status:** stage placeholder.

## Likely tasks

### T14.2.1.1 — Toolchain — MSVC vs mingw

Pick: MSVC + vcpkg (most native, slowest fight against C++23
features depending on MSVC version) OR mingw + msys2 (closer
to Linux, may have soundcard quirks). Document the choice.

### T14.2.1.2 — System dep gathering

Many of the Linux deps are Linux-specific (PipeWire,
pulseaudio, sd-bus). Replace with Windows equivalents:
WASAPI for audio, no sd-bus (sd-bus is gated already
behind ACVA_ENABLE_SDBUS).

### T14.2.1.3 — Service registration

Windows Service Control Manager replacement for systemd.
`packaging/windows/install.ps1` registers acva as a service.

### T14.2.1.4 — Compose on Windows

WSL2 + Docker Desktop is the dev path. Operators get the
same containers; only acva itself changes.

### T14.2.1.5 — Audio backend

PortAudio + WASAPI on Windows. AEC equivalent: WASAPI's
voice mode flag — much weaker than PipeWire's module-echo-
cancel; in-process WebRTC APM probably becomes the default
on Windows.
