# S14.4.2 — All-in-one C++ executable

**Phase:** P14.4 (cross-platform packaging)
**Estimate:** 1-2 weeks (investigation; impl deferred per Q2)
**Status:** stage placeholder.
**Shape:** investigation, operationalises Q2.

## Context

`ideas.md` line 93 + line 137: "all-in-one C++ app
(llama.cpp + whisper.cpp + others, libs instead of servers)"
and "??? single all-in-one executable, run as single
process".

`Q2` parks this with "default rejected — violates pillar #1
(process isolation for backends)". Q2 also says "decide by
M14 cross-platform" — this is that stage.

The cross-platform context is what changes the calculus:
on macOS / Windows, the multi-process Compose model is
much more friction than on Linux. An all-in-one binary
would simplify packaging dramatically.

## Tasks

### T14.4.2.1 — Library-mode survey

For each backend, what's the runtime-library shape?

- **libllama** — yes; first-class (it's how llama-server
  is built). C ABI.
- **libwhisper** — yes; same upstream as the server.
- **Piper / Kokoro** — Python today; would need a C/C++
  inference path or native ONNX.
- **Speaches** — Python; not viable as a library.

Speaches dependency is the killer for full all-in-one.

### T14.4.2.2 — Two-tier proposal

If true all-in-one is infeasible because of Speaches, a
*partial* all-in-one is still valuable:
- libllama embedded, no llama-server.
- libwhisper embedded, no streaming Speaches/STT.
- Piper-direct (C++) embedded for non-streaming TTS.
- All-in-one binary with the runtime models bundled (or
  registry-pulled).

This loses streaming (M5) and OpenAI compatibility but
gains single-binary install.

### T14.4.2.3 — Build-flag opt-in

`-DACVA_ALL_IN_ONE=ON` — enables libllama + libwhisper +
Piper-direct. CI builds both shapes; default is multi-
process.

### T14.4.2.4 — Pillar discussion

Pillar #1 says "do not embed model runtimes until post-M8".
We're post-M8. The pillar is a guide not a law. Re-evaluate
in this stage's write-up; if the answer is "guide still
holds", document the trigger condition that would flip it.
