# S14.1.1 — macOS Metal port kickoff

**Phase:** P14.1 (macOS Metal port)
**Estimate:** 4-5 weeks
**Status:** stage placeholder.

## Likely tasks

### T14.1.1.1 — Toolchain + dep audit

Verify each system dep has a macOS equivalent. Most do
(homebrew has spdlog, fmt, prometheus-cpp, glaze,
opentelemetry-cpp, libdatachannel). Speaches runs in Docker
Desktop. Onnxruntime + WebRTC APM + libdatachannel may need
homebrew or source builds.

### T14.1.1.2 — LLM backend swap

llama.cpp's Metal backend replaces CUDA. The
model-controller sidecar currently assumes Docker on Linux;
on macOS, llama-server runs natively (homebrew /
self-built). M11 model-controller likely needs a macOS-
native variant.

### T14.1.1.3 — Audio backend

PortAudio works on macOS via CoreAudio. AEC: in-process
WebRTC APM works; CoreAudio voice processing IO unit is the
"system AEC" equivalent of PipeWire's module-echo-cancel
on Linux.

### T14.1.1.4 — Distinct builds

`CMakePresets.json` gets `mac-dev` / `mac-release` presets.
Conditional source compilation for OS-specific paths
(`#ifdef __APPLE__`).

### T14.1.1.5 — Compose alternative

Docker Desktop on Mac is heavier than Linux. Document the
all-native path: llama-server + Speaches both run as
homebrew daemons; acva connects to them.
