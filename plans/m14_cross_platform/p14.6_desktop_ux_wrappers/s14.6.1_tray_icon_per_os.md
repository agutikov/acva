# S14.6.1 — Tray icon per OS

**Phase:** P14.6 (desktop UX wrappers)
**Estimate:** 1-2 weeks (4-5 days per OS)
**Status:** stage placeholder.
**Shape:** investigation + impl.

## Context

`ideas.md` line 140: "background service + tray icon with
quick small menu + web-ui".

A small tray icon shows acva's state (Listening /
Speaking / Muted) and exposes a quick menu (mute, new
session, open Web UI, restart). The Web UI does the heavy
lifting; tray is the discoverable surface.

## Tasks

### T14.6.1.1 — Linux tray (libappindicator / KStatusNotifier)

Most desktops support StatusNotifierItem (KDE) with
libappindicator (GTK fallback). Build a tiny C++/Qt or
gtkmm tray binary that shells to the acva HTTP control
plane.

### T14.6.1.2 — macOS tray (NSStatusItem)

Cocoa NSStatusItem in a small Swift / Obj-C++ binary.
Same control-plane HTTP interaction.

### T14.6.1.3 — Windows tray (Shell_NotifyIcon)

Win32 Shell_NotifyIcon; small wrapper.

### T14.6.1.4 — Shared control surface

All three trays speak the same `/status` + control HTTP.
The tray is a thin client; if Web UI is open the tray's
state mirrors the page.

### T14.6.1.5 — Acceptance

On each OS that has its tray shipped, click "mute" in tray
→ acva muted within 200 ms; status icon updates.

## Resolves / depends

- Feeds Q28 (wrapper choice).
- Linux part can ship independently of M14.P14.1 / P14.2.
