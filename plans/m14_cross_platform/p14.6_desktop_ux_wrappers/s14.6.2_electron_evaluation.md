# S14.6.2 — Electron evaluation

**Phase:** P14.6 (desktop UX wrappers)
**Estimate:** 4-5 days
**Status:** stage placeholder.
**Shape:** investigation only.

## Context

`ideas.md` line 141: "desktop app with electron-based UI,
loads server web-ui ???".

Electron wraps the M10 Web UI in an OS-window with
auto-updates, deep-links, OS notifications, etc. Cost: ~200
MB binary, ~150 MB RAM idle, security surface (Chromium
embedded), JS toolchain in the build.

## Tasks

### T14.6.2.1 — Survey

What does Electron buy *over* tray + browser?
- auto-update across OSes
- consistent window chrome
- deep links / URL handler registration
- OS-native notifications
- file pickers, native menus

What's the cost?
- ~200 MB binary footprint per OS
- Chromium security tracking
- electron-builder + signing
- second toolchain

### T14.6.2.2 — Prototype size check

Build a hello-world Electron app that loads the Web UI;
measure final binary + idle RAM.

### T14.6.2.3 — Decision

Closure note in this stage. Likely outcome: "tray + browser
is enough for v1; Electron deferred until Web UI scope
makes a desktop wrapper compelling".
