# S14.6.4 — Quick-menu UX

**Phase:** P14.6 (desktop UX wrappers)
**Estimate:** 3-4 days
**Status:** stage placeholder.
**Shape:** UX design + impl.

## Context

`ideas.md` line 140: "tray icon with quick small menu".

Whatever wrapper lands (tray-only or Electron / Tauri),
the menu surface is the same:

```
acva 🟢 Listening
─────────────────
🔇 Mute / Unmute
📝 New session
🌐 Open Web UI
⚙ Settings
🔁 Restart
─────────────────
Quit
```

The small surface is what the operator sees most often;
worth designing it deliberately.

## Tasks

### T14.6.4.1 — Menu items + intent

Land the canonical menu list. Each item maps to one
control endpoint:
- Mute → `POST /mute`
- Unmute → `POST /unmute`
- New session → `POST /new-session`
- Open Web UI → browser launcher
- Settings → opens UI's settings tab
- Restart → `POST /restart`
- Quit → tray exit (acva keeps running unless a
  separate "Stop service" is offered).

### T14.6.4.2 — Status icon iconography

Distinct icons per FSM state (Listening / Speaking /
Thinking / Muted / Error). Operator-recognisable at
glance. Resolution-independent SVG sources.

### T14.6.4.3 — Notifications

Optional toast on key transitions: "Speaches went down
— remediation queued"; "Stuck > 90s — auto-restarting".
Tied to existing M8B watchdog signals.

### T14.6.4.4 — Acceptance

Operator dogfood: 1 hour with the tray as the only UI
surface. List rough edges; iterate.

## Resolves / depends

- Lands the menu surface for whichever wrapper Q28 picks.
- Pairs with M8A control endpoints + M8B watchdog
  signals.
