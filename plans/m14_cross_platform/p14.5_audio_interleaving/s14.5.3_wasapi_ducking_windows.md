# S14.5.3 — WASAPI ducking on Windows

**Phase:** P14.5 (audio interleaving)
**Estimate:** 4-5 days
**Status:** stage placeholder.
**Shape:** investigation + impl, gated on M14.P14.2.

## Context

`ideas.md` line 95 + Windows port. WASAPI exposes
`AUDCLNT_STREAMFLAGS_AUTOCONVERTPCM` and stream categories
(`AudioCategory_Communications`, `AudioCategory_Movie`)
that drive Windows' built-in ducking.

## Tasks

### T14.5.3.1 — Stream category survey

When acva opens its WASAPI playback stream, set
`AudioClientProperties.eCategory = AudioCategory_
Communications`. Windows then ducks media streams while
acva plays. Standard pattern from telephony / chat apps.

### T14.5.3.2 — Implementation

Tie the category to `cfg.audio.ducking.enabled`. When
disabled, use `AudioCategory_Other` (no ducking).

### T14.5.3.3 — Acceptance

Spotify on Windows ducks when acva speaks; restores after.
