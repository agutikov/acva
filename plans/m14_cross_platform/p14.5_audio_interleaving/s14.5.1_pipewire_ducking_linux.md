# S14.5.1 — PipeWire ducking on Linux

**Phase:** P14.5 (audio interleaving)
**Estimate:** 4-5 days
**Status:** stage placeholder.
**Shape:** investigation + impl.

## Context

`ideas.md` line 95: "Interleaving with players (music,
browser, etc)". Linux first because acva's primary OS is
Linux + PipeWire (M6B already uses `module-echo-cancel`).

PipeWire has stream priorities and `media.role`-based
ducking via wireplumber rules; the question is which
mechanism gives operators a clean opt-in.

## Tasks

### T14.5.1.1 — Mechanism survey

Three approaches:
- **wireplumber rule** — declare `media.role=Notification`
  on the acva playback stream; wireplumber's default rules
  duck other streams. Cleanest; depends on operator's
  wireplumber version.
- **manual `pactl set-sink-input-volume`** — duck other
  streams imperatively when acva starts speaking, restore
  on PlaybackFinished. Reliable but brittle to detect
  "other streams".
- **`module-role-ducking`** — explicit module load.
  Predictable, requires module load at startup.

### T14.5.1.2 — Implementation

Pick the wireplumber-rule path; fall back to imperative
if rule doesn't apply on operator's distro. Ship a
wireplumber rule snippet in `packaging/`.

### T14.5.1.3 — Config surface

`cfg.audio.ducking.{enabled, amount_db, attack_ms,
release_ms}`. Hot-reloadable.

### T14.5.1.4 — Acceptance

Music playing in Firefox; acva starts speaking → music
ducks by configured amount within 100 ms; restores
within 200 ms after PlaybackFinished.

## Resolves / depends

- Feeds Q27 (interleaving policy).
- Pairs with M6B PipeWire integration (same audio stack).
