# S15.2.1 — Extract reference modules

**Phase:** P15.2 (extract reference modules)
**Estimate:** 3-4 weeks
**Status:** stage placeholder.

## Tasks

### T15.2.1.1 — Pick first extraction targets

Audio backend (PortAudio wrapper) is the cleanest first
extraction — well-bounded, clear interface in M4. Memory
store (SQLite-based MemoryThread) is harder due to the
WAL coexistence rules but worth it.

### T15.2.1.2 — Refactor to the ABI

Each extracted subsystem becomes a separate target in
CMake building a .so. The acva binary links against the
ABI, not the implementation; the implementation .so loads
at runtime.

### T15.2.1.3 — Default module set

The shipped modules become the M0–M14 default behavior.
Operators who want plain acva get exactly what they had;
operators who want a custom module replace the relevant
.so.

### T15.2.1.4 — Bench

Module call boundary vs direct call: ≤ 1 µs overhead per
call on the dev box. Audio path's hot loop must not regress.
