# S13.2.1 — Flatpak / AppImage self-contained installers

**Phase:** P13.2 (self-contained installers)
**Estimate:** 3-4 weeks
**Status:** stage placeholder.

## Tasks

### T13.2.1.1 — Flatpak

`packaging/flatpak/com.github.acva.json` manifest. Bundles
acva + the model-controller + systemd-equivalent service
files. Sandboxes via Flatpak; needs explicit permissions
for audio (`--device=all`), GPU (`--device=dri`), models
dir.

### T13.2.1.2 — AppImage

`packaging/appimage/build.sh`. Single executable bundling
all deps. Larger but simpler — no host install at all.

### T13.2.1.3 — Compose-stack inclusion?

Open question: do these installers bundle the Compose
stack (llama + speaches) or just acva? Probably just acva;
backends stay external for VRAM / model-update reasons.
Document the decision.

### T13.2.1.4 — Snap (skipped unless demand)

Snap support is more friction than value for the dev OS
(Manjaro/Arch). Punt unless an operator on Ubuntu asks.
