# S13.3.1 — CI/CD + release engineering

**Phase:** P13.3 (CI/CD + release engineering)
**Estimate:** 2-3 weeks
**Status:** stage placeholder.

## Tasks

### T13.3.1.1 — GitHub Actions: build + test

Matrix: Linux x86_64 with the system deps from README.md.
Build + run unit tests on every push to main. Skip
integration tests in CI (need live Speaches; runs locally).

### T13.3.1.2 — Release tagging

`scripts/release.sh` — tag, build artifacts (deb, rpm,
flatpak, appimage), generate changelog from commit
messages, create GitHub release.

### T13.3.1.3 — Artifact signing

GPG sign release artifacts. Public key in repo + on a
keyserver.

### T13.3.1.4 — gh-pages docs publish

`docs/` rendered via mkdocs or similar, deployed to
gh-pages on each tag. Public URL: docs.acva.something.

### T13.3.1.5 — Smoke test post-release

CI job that downloads the just-released artifacts, runs
`acva demo health` against a stub Compose stack, fails
the release if smoke fails.
