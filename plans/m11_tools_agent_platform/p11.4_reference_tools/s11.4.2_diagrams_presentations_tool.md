# S11.4.2 — Diagrams + presentations tool

**Phase:** P11.4 (reference tools)
**Estimate:** 1 week
**Status:** stage placeholder.
**Shape:** impl.

## Context

`ideas.md` lines 82: "diagrams and presentations".
Promoted from "optional" task (`s11.4.1` T11.4.1.4) into a
dedicated stage so it doesn't get deprioritised silently.

The user can ask "draw me a system diagram of acva"; the
LLM emits a `tool_call(diagram, mermaid_source)`; the tool
renders to PNG/SVG and either pushes to the M10 Web UI or
saves to a known path.

## Tasks

### T11.4.2.1 — Mermaid renderer

Subprocess wrapper around `mermaid-cli` (npx mmdc). Input:
mermaid source. Output: SVG + PNG. Cached by content-hash.

### T11.4.2.2 — Graphviz fallback

For LLMs that prefer DOT syntax, accept DOT input; render
via `dot` subprocess.

### T11.4.2.3 — Presentations

Marp / reveal.js source as input → static HTML deck.
Output served by the M10 Web UI as a routed asset.

### T11.4.2.4 — UX wiring

Diagram tool result is announced verbally ("I've drawn
a diagram") and a link / image goes into the M10 chat
trace. No TTS of the SVG content.

### T11.4.2.5 — Acceptance

Operator asks "show me the threading model" → LLM
generates mermaid → diagram visible in Web UI within
5 s of the LLM finishing.

## Resolves / depends

- Depends on M11.P11.1 framework + M11.P11.2 tool-call
  grammar.
- Pairs with M10.P10.3 chat trace UI.
