# S11.1.1 — Tool framework kickoff

**Phase:** P11.1 (tool framework)  
**Estimate:** 3-4 weeks (split into 3-4 stages when picked up)  
**Status:** stage placeholder.  
**Open question:** Q6 (sandbox model) — decided here.

## Likely tasks (split into stages)

### T11.1.1.1 — Manifest schema

`tool.yaml` per tool: name, version, description,
input/output JSON schema, permissions (read/write paths,
network, env), runtime (subprocess/WASM/native).

### T11.1.1.2 — Lifecycle

Spawn → call → result → cleanup. Long-running tools
return an op id; status polled via
`GET /api/v1/tool-calls/{id}`. Cancellation propagates from
turn cancel.

### T11.1.1.3 — Sandbox decision (Q6)

Pick subprocess (default), WASM (wasmtime — opt-in), or
native (M15-gated). Document in `m11_tools_agent_platform/open_questions.md`.

### T11.1.1.4 — Permission model

Per-tool capabilities expressed in the manifest, validated
at install time, enforced at call time.

### T11.1.1.5 — Tests

Unit: manifest parsing, permission validation, mocked
lifecycle. Integration: a hello-world subprocess tool,
end-to-end call from a synthetic LLM tool_call action.
