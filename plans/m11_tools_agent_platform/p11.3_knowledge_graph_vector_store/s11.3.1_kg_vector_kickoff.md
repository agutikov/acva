# S11.3.1 — Knowledge graph + vector store kickoff

**Phase:** P11.3 (knowledge graph + vector store)  
**Estimate:** 3-4 weeks  
**Status:** stage placeholder.  
**Open question:** Q7 (KG storage) — decided here.

## Likely tasks

### T11.3.1.1 — Storage decision (Q7)

SQLite + sqlite-vec (recommended baseline, lowest dep
addition) vs sibling vector DB (qdrant, chroma) vs graph DB
(neo4j, kuzu). Document in `m11_tools_agent_platform/open_questions.md`.

### T11.3.1.2 — Schema design

Entities, relations, attributes. Per-entity embedding cache
in the vector store. Per-session scoping (multi-user comes
in M12).

### T11.3.1.3 — Embedding pipeline

Pick a small embedding model (BGE-small or similar, ~100 MB
ONNX). Run on the existing Speaches container or as a fresh
ONNX-runtime path. Per-utterance + per-fact embedding on
write.

### T11.3.1.4 — Retrieval API

`KnowledgeStore::retrieve(query, k)` → list of facts +
relevance scores. Wired into `PromptBuilder` as an optional
context layer.

### T11.3.1.5 — Memory schema migration

Existing `facts` table from M1 stays; new `entities` +
`relations` + `embeddings` tables. Migration on startup
(first M11 run).
