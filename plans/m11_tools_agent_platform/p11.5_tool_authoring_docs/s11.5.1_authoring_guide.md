# S11.5.1 — Tool authoring guide + sandbox revisit

**Phase:** P11.5 (tool authoring + docs)

**Status:** stage placeholder.

## Likely tasks

### T11.5.1.1 — Authoring guide

`docs/guide/tool-authoring.md`: manifest reference, lifecycle
contract, permission model, sandbox model, examples from
P11.4.

### T11.5.1.2 — Sandbox revisit (Q6 follow-up)

If subprocess started as the baseline, re-evaluate after
P11.4 dogfood. Is WASM worth the build dep? Document
findings.

### T11.5.1.3 — Permission UX in M10 Web UI

Operators install a tool → UI surfaces the requested
permissions, asks for explicit confirmation per category
(filesystem read/write, network, env access). Permissions
persist in a per-tool grant file.

### T11.5.1.4 — Tool registry stub (optional)

A simple `tools/acva-tools install <git-url>` flow that
clones a tool repo, validates the manifest, runs through
permission prompt. Distinguishes from
`tools/acva-models` (models = ML weights, tools = code).

### T11.5.1.5 — Acceptance

A teammate (not the original author) writes a hello-world
tool following the guide, ships it through `acva-tools
install`, calls it from the assistant, sees the result.
That's the milestone-closing test.
