# S9.0.1 — Fix "no metrics in Grafana / Prometheus"

**Phase:** P9.0 (MVP bug sweep)  
**Estimate:** 1-2 days  
**Source bug:** [`plans/bugs/no_metrics_in_grafana.md`](../../bugs/no_metrics_in_grafana.md)

## Context

Operators bring up `packaging/observability/` and see blank Grafana
panels. Root cause: the dashboard scrapes acva on
`127.0.0.1:9876/metrics`; if acva isn't running, every `voice_*`
panel reads "No data" and there's no obvious indicator why.

Fix is two-part: surface the prereq in docs so operators know to
start acva, *and* make the dashboard self-explain when the target
is down.

## Acceptance

- README + `docs/guide/operations.md` § Observability stack carry
  a bold prereq line: *"Grafana panels populate from acva's
  `/metrics` endpoint — start acva before opening the dashboard."*
- Dashboard top row has a "service health" panel that surfaces
  Prometheus's `up{instance="127.0.0.1:9876"}` and
  `up{instance="127.0.0.1:8081"}` so a single glance shows
  whether each scrape target is up.
- Optional: panels using `voice_*` carry a Grafana `noValue:
  "acva not running — start it and reload"` text so empty panels
  show the explanation rather than just "No data".
