# S9.0.3 — Model-controller visibility + smoke

**Phase:** P9.0 (MVP bug sweep)  
**Estimate:** 2-3 days  
**Source bug:** [`plans/bugs/model_controller_unverified.md`](../../bugs/model_controller_unverified.md)

## Context

The M8A model-controller sidecar (`packaging/model-controller/`)
was code-complete but explicitly **never end-to-end smoke-tested**
(per CLAUDE.md M8A Step 5). It's also not auto-wired into Compose
— operators must paste in the README stanza. Today there's no
indicator from the operator side whether the sidecar is reachable,
healthy, or talking to acva.

Two-part fix: run the smoke pass (does it work?), and surface
state to operators (`/status` block + Grafana panel + metrics).

## Acceptance

- End-to-end smoke passes: from a clean state with two GGUFs
  installed, change `cfg.llm.model_file`, restart acva, verify
  the sidecar swaps the llama container and acva picks up the
  new model. Document the smoke procedure in
  `packaging/model-controller/README.md`.
- `GET /status` includes a `model_controller` block when the
  feature is enabled (any non-empty `cfg.llm.model_controller_url`),
  with `reachable`, `loaded_model`, `requested_model`,
  `in_progress`, `last_swap_at`, `last_swap_duration_ms`. When
  unreachable, surface a remediation hint similar to the
  Speaches wedge classifier.
- Metrics: `voice_model_controller_reachable{}`,
  `voice_model_controller_swap_total{outcome}`,
  `voice_model_controller_swap_duration_seconds` histogram.
- Optional: Grafana panel showing the current loaded model +
  swap-in-progress indicator.

## Tasks

### Task T9.0.3.1 — Run the smoke pass

Wire the sidecar into `packaging/compose/docker-compose.yml`
(behind a Compose profile so it's still opt-in but not
copy-paste). Two GGUFs installed (e.g. `dialog` + `socratic`),
edit `~/.config/acva/default.yaml` to switch the alias, restart
acva, observe the swap. Record any failure modes — these
become P9.0 sub-tasks if non-trivial.

### Task T9.0.3.2 — `/status` block

Extend `src/orchestrator/observability/status_extra.{hpp,cpp}`
to include a `model_controller` JSON block. Read the current
state from a new `ModelControllerStatus` snapshot the
`ModelControllerClient` updates on each call.

The block:

```json
{
  "model_controller": {
    "url": "http://127.0.0.1:9877",
    "reachable": true,
    "loaded_model": "Qwen2.5-7B-Instruct-Q4_K_M.gguf",
    "requested_model": "Qwen2.5-7B-Instruct-Q4_K_M.gguf",
    "in_progress": false,
    "last_swap_at": "2026-05-08T14:31:22Z",
    "last_swap_duration_ms": 4231,
    "remediation": null
  }
}
```

When `reachable: false`, set `remediation` to a one-liner
pointing at the `packaging/model-controller/README.md` install
stanza.

### Task T9.0.3.3 — Metrics

Register three new Prometheus series in
`src/metrics/registry.{hpp,cpp}`:

- `voice_model_controller_reachable` (gauge, 0/1)
- `voice_model_controller_swap_total{outcome="ok|failed|skipped"}`
  (counter)
- `voice_model_controller_swap_duration_seconds` (histogram,
  buckets 0.5/1/2/5/10/30/60)

Wire from `ModelControllerClient` calls. Add a small Grafana
panel to `packaging/grafana/acva.json` showing
`voice_model_controller_reachable` as a stat + the swap
histogram as a heatmap.

### Task T9.0.3.4 — Doc the smoke procedure

Append to `packaging/model-controller/README.md` a § "Smoke
test" with the steps from T9.0.3.1, so future operators (or
future Claude) can re-verify after upstream Docker / llama
changes.
