# S9.0.2 — Wake-word threshold + warm-up UX

**Phase:** P9.0 (MVP bug sweep)  
**Estimate:** 1-2 days  
**Source bug:** [`plans/bugs/wake_word_latency_and_threshold.md`](../../bugs/wake_word_latency_and_threshold.md)

## Context

The openWakeWord stock `hey-jarvis` classifier peaks 0.37–0.47 on
Russian-accented speech. Default threshold is 0.55 / 0.6 / 0.75
across personalities — strict-mode operators never trigger.
Separately, the 2.6 s warm-up window prints `score=0.000` per tick
in `acva demo wake-word`, which looks like the model is broken.

Wake-word ships off-by-default
([§L8 in the MVP archive](../../../docs/history/MVP/open_questions.md))
so this isn't load-bearing for the conversational-mode product
line — but operators who opt in to sleep-and-wake mode hit it
immediately.

## Acceptance

- Default threshold lowered to 0.45 (was 0.55 at the top level;
  personality overrides preserved). Verified by scoring the
  reference fixture: peaks crossing the new threshold, white
  noise + silence stay below it.
- `acva demo wake-word` differentiates warm-up from live ticks:
  first ~26 ticks print `t=…s warm-up` instead of `score=0.000`.
- `docs/guide/operations.md` gains a § "Wake-word calibration"
  with the recommended workflow: run `acva demo wake-word
  --duration 30`, observe peak, set threshold to peak × 0.85.

## Tasks

### Task T9.0.2.1 — Lower default threshold

Edit `config/default.yaml` `audio.wake_word.threshold: 0.55 →
0.45`. Verify against the offline fixture
(`scripts/wake-word-offline.sh`) that peaks cross the new
threshold for the test recording, and white noise + silence
stay < 0.05.

### Task T9.0.2.2 — Warm-up indicator in the demo

Edit `src/demos/wake_word.cpp` to query
`WakeWord::is_warmed_up()` (new method on `audio::WakeWord`,
returns true once the embedding window has filled). Print
`t=…s warm-up` until warmed; switch to score-printing after.

The 2.6 s warm-up duration is documented in
`src/audio/wake_word.hpp`; make `is_warmed_up()` deterministic
(returns true iff `embeddings.size() >= kClassifierWindow`).

### Task T9.0.2.3 — Calibration doc

Add a short § "Wake-word calibration" to
`docs/guide/operations.md` § Switching models & personalities.
Recommended workflow:

```sh
# 1. Set threshold to 0.0 temporarily so every tick prints
# 2. Speak the phrase 5 times in your normal voice
./_build/dev/acva demo wake-word --duration 30
# 3. Note the peak score
# 4. Set threshold to peak × 0.85 in config/default.yaml
# 5. Hot-reload (audio.wake_word.threshold is hot, no restart)
curl -fsS -X POST http://127.0.0.1:9876/reload
```
