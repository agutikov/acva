# S9.5.7 — Style post-processing pipeline

**Phase:** P9.5 (TTS expression)
**Estimate:** 4-6 days
**Status:** stage placeholder.
**Shape:** investigation + impl.

## Context

`ideas.md` line 126: "style post processing".

Distinct from S9.5.1 (cheap DSP on the audio stream): this
stage processes the *text* before TTS to inject persona-
appropriate phrasing tweaks — contractions, intensifiers,
disfluency markers, regional spelling.

Example: the same LLM reply
  "I do not know."
becomes, depending on persona overlay:
- `consultant`: "I don't know."
- `casual`: "Beats me."
- `theatrical`: "Alas — I haven't the faintest."

## Tasks

### T9.5.7.1 — Position in pipeline

Place between the SentenceSplitter and TtsBridge. Operates
on `event::LlmSentence` content; emits a transformed
sentence on a new event or in-place mutation.

### T9.5.7.2 — Transform library

`config/text_styles/*.yaml` — declarative regex / phrase
replace + light tag-aware substitution. Don't ship a
transformer-based rewrite for v1; that's too heavy and too
slow.

### T9.5.7.3 — Per-personality wiring

Personality overlay names an active style; orchestrator
loads the rules at boot and on warm-restart.

### T9.5.7.4 — Tests

Idempotent: applying the same style twice produces the same
output. No-op style is a real no-op (zero allocs).

### T9.5.7.5 — Risk: it can break factual content

The style can rewrite "I do not know" but must not rewrite
"I do not have the password" or anything that flips
semantic meaning. Document the discipline; any rule that
might cross that line goes through manual review.

## Resolves / depends

- Pairs with S9.5.1 audio DSP and S9.5.6 mood mapping —
  text style is the third lane in the same theme.
- No new question.
