# S9.5.8 — Min vs max preset comparison harness

**Phase:** P9.5 (TTS expression)
**Estimate:** 3-4 days
**Status:** stage placeholder.
**Shape:** harness + UX deliverable. Closes P9.5.

## Context

`ideas.md` lines 73-75:

> compare:
>     - min default config: no humanization, plain direct question-answer, default computer voice
>     - max human-like config: voice, drift, background thinking, etc...

This is the gate that turns the rest of P9.5 from "we built
features" into "operators can pick a stance and hear the
difference". Two presets ship; the harness produces a side-
by-side audio + structured rating CSV.

## Tasks

### T9.5.8.1 — Two presets

`config/personality/preset-min.yaml` and `preset-max.yaml`:

- **min**: default voice, no DSP, no RVC, neutral prosody,
  no style post, no verbal thinking, terse system prompt.
- **max**: subtle-mic DSP, RVC if available, mood-aware
  prosody, casual style overlay, verbal thinking opt-in.

### T9.5.8.2 — A/B harness

`scripts/preset-ab.py` — fires the same prompt set through
both presets, captures audio + metrics, lays out a
side-by-side directory tree. Rater fills out a CSV with
1-5 scores on naturalness / clarity / engagement.

### T9.5.8.3 — Operator dogfood

Run on at least the soak-driver prompt set + 10 free-form
prompts. Capture in `docs/reports/preset_comparison.md`.

### T9.5.8.4 — Documentation

`docs/guide/voice-presets.md` — operator guide to the two
presets, what each enables, how to mix.

## Resolves / depends

- Closes P9.5 acceptance #1.
- Depends on whichever subset of S9.5.1-7 actually ships;
  the presets ship even if some features defer (the
  preset just doesn't enable them).
