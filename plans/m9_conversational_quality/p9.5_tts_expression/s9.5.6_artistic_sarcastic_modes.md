# S9.5.6 — Artistic / sarcastic / expressive modes

**Phase:** P9.5 (TTS expression)
**Estimate:** 1 week
**Status:** stage placeholder.
**Shape:** impl + UX investigation.

## Context

`ideas.md` lines 69-71: "artistic speaking with expression",
"sarcastic".

Beyond plain DSP and prosody, the operator wants the assistant
to read at full volume some content and dryly, with sarcasm,
read other content. This is partly LLM-side (the model needs
to mark intent) and partly TTS-side (the TTS needs to
respond).

## Tasks

### T9.5.6.1 — Action grammar extension (Q19)

Sarcasm + expression are LLM-side hints. Extend S9.4.3's
GBNF actions:

```
speak(content, mood?)
mood ::= "neutral" | "sarcastic" | "excited" | "warm" | …
```

Q19 (open question) decides the full mood vocabulary.

### T9.5.6.2 — Mood → prosody mapping

Each mood maps to a prosody preset (S9.5.5) + DSP filter
selection (S9.5.1). E.g. `sarcastic` → flatter pitch + slight
delay between words; `excited` → faster rate + bright EQ.
Mappings live in `config/moods/*.yaml`.

### T9.5.6.3 — Voice selection per mood

Some moods may want a different *voice* (e.g. an "evil"
mood for narrative; a "child" voice for a story). When a
mood maps to a voice not in the registry, fall back to
prosody-only.

### T9.5.6.4 — Acceptance

Operator A/B: same content rendered in 4 moods. Listener
ID's the mood ≥ 70% of the time blind.

## Resolves / depends

- Touches Q19 (action superset).
- Depends on S9.5.1 + S9.5.5 (prosody + DSP).
- Pairs with S9.5.3 RVC if landed (RVC enables full voice
  swap; otherwise mood = prosody preset only).
