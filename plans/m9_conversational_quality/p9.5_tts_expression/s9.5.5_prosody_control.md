# S9.5.5 — Prosody control (rate / pitch / emphasis)

**Phase:** P9.5 (TTS expression)
**Estimate:** 4-5 days
**Status:** stage placeholder.
**Shape:** investigation + impl.

## Context

`ideas.md` lines 65-75 (sound filters, expression, comparison).

Prosody knobs the operator wants:
- **rate** — words per minute (existing `cfg.tts.tempo_wpm`).
- **pitch** — semitone shift.
- **emphasis** — per-word stress hints.
- **pause** — explicit `<break>` between sentences.

S9.5.2 produces the SSML support matrix; this stage decides
which knobs we expose via config / hot-reload, and which need
post-synthesis DSP (S9.5.1) to fake.

## Tasks

### T9.5.5.1 — Knob inventory + control surface

Add `cfg.tts.prosody.{rate_wpm, pitch_semitones,
emphasis_strength}` to config. Hot-reloadable. Default
identity (today's behavior).

### T9.5.5.2 — Backend pass-through

For knobs the backend supports natively (e.g. Kokoro's
`speed`), pass through. For knobs it doesn't (pitch on
Piper), apply post-synthesis DSP via the S9.5.1 chain.

### T9.5.5.3 — Per-personality overrides

Personality overlays already cover other config; this stage
extends overlays to prosody. Default Russian-language
prosody might use a slightly different rate from English.

### T9.5.5.4 — Tests

Unit: PromptBuilder + TtsBridge thread the prosody block
through to the request body. Integration: round-trip on
fixture audio confirms the rate change is audible (length
ratio in ±5% of expected).

## Resolves / depends

- Depends on S9.5.1 (DSP chain) for knobs the backend
  doesn't support natively.
- Pairs with S9.5.2 (SSML) — pass-through knobs land here.
