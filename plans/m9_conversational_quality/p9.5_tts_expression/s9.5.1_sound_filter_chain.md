# S9.5.1 — Sound filter chain (cheap DSP)

**Phase:** P9.5 (TTS expression)
**Estimate:** 4-6 days
**Status:** stage placeholder.
**Shape:** investigation + impl.

## Context

`ideas.md` line 65: "Sound filters cheap way to make voice
more natural".

Before any neural voice work, run cheap classical DSP on the
TTS output: light reverb, EQ tilt, subtle noise floor, slight
compression. Each is < 1% CPU and can transform a clinical TTS
into something that sounds like it came from a microphone.

## Tasks

### T9.5.1.1 — Filter library survey

Two candidates:
- **SoX** subprocess — well-known, easy to script, but
  per-chunk subprocess overhead is bad for streaming.
- **In-process** via a small wrapper around an open DSP
  lib (libsamplerate already in tree; explore
  `kissfft` / `julius`). Cheaper at runtime, more code.

The realtime audio path is sacred (pillar #2) — whatever
ships must not allocate or block on the playback thread.

### T9.5.1.2 — Operator preset library

`config/audio_filters/*.yaml` — named presets:
`subtle-mic`, `phone-call`, `radio-warm`, `bright`,
`hifi`. Each has a chain of filters with knobs.

### T9.5.1.3 — Wiring

Insert between `Resampler` (TTS-side) and `PlaybackQueue`.
New stage `audio::TtsFilterChain` reads
`cfg.tts.filter_chain` (alias into the preset library or
inline definition).

### T9.5.1.4 — Acceptance

Operator A/B: same Speaches voice with filter `none` vs
`subtle-mic`. Captured into `docs/reports/` as part of
S9.5.8 harness data.
