# S9.4.11 — Temperature + sampling per personality

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 2-3 days
**Status:** stage placeholder.
**Shape:** config-surface task.

## Context

`ideas.md` line 124: "temperature".

Today `cfg.llm.temperature` is a single global hot-reload field
(M8A). Different personalities want different sampling defaults
— a `consultant` should be conservative (temp ~0.3); a
`storyteller` benefits from higher (~0.9). Personality overlays
already exist as M8C config-precedence; this stage extends
overlays to cover sampling.

## Tasks

### T9.4.11.1 — Config schema

Personality overlay can set `llm.{temperature, top_p, top_k,
repeat_penalty}`. Falls back to the global defaults when the
overlay omits a field. Validators ensure sensible ranges.

### T9.4.11.2 — Wiring

`PromptBuilder` (or `LlmClient::submit`) reads the active
personality's effective sampling block per turn.

### T9.4.11.3 — Tests

Personality switch via warm-restart picks up the overlay
sampling. Unit test: two personalities with different temps
produce different outgoing-request bodies on the same prompt.
