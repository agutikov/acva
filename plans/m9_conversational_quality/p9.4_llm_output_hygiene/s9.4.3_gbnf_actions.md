# S9.4.3 — GBNF: explicit {think | wait | speak | ignore} actions

**Phase:** P9.4 (LLM output hygiene)  
**Estimate:** 4-5 days  
**Forward link:** the GBNF infrastructure here gets reused by
M11.P11.2 (tool-call structured output).

## Context

Today the LLM outputs free-form text, which the SentenceSplitter
shovels into TTS. The model has no way to say "let me think for
a second" without speaking those words; no way to indicate
"I want to ignore this utterance"; and no way to say "wait — I
need information before I can answer."

GBNF (Grammar-Based Normal Form) at the llama.cpp level lets us
constrain the model's output to a small action grammar. The
Manager interprets each action and routes appropriately.

## Action set

```
think  — internal reasoning, never spoken (logs at debug, optionally
         shown in Web UI when M10 lands)
wait   — explicit "don't speak" decision; FSM stays in Listening
         after this turn rather than firing a TTS
speak  — natural language to be voiced; identical to today's
         output
ignore — like wait, but also marks the user's preceding turn as
         not-meaningful (e.g. cough, throat-clear)
```

A reply can chain: `think("user wants the time") → speak("It's
3 PM").` Multiple speaks chain into a longer answer. A standalone
`wait` or `ignore` produces no speech.

## Tasks

### Task T9.4.3.1 — Grammar definition

`src/llm/grammar.{hpp,cpp}` (new). Embeds a GBNF string:

```
root      ::= (action " " | action)+
action    ::= think | wait | speak | ignore
think     ::= "think(\"" content "\")"
wait      ::= "wait()"
speak     ::= "speak(\"" content "\")"
ignore    ::= "ignore()"
content   ::= ([^"\\] | "\\\"" | "\\\\")*
```

Tunable: `cfg.llm.grammar.actions_enabled` (default true).
When off, M9 ships free-form output (M5 behavior).

### Task T9.4.3.2 — llama-server wiring

llama.cpp's chat-completions API supports `grammar` in the
request body. `LlmClient::submit` accepts a grammar string;
when set, the model output respects the grammar.

Edge case: streaming + grammar. Tokens still stream; the parser
in T9.4.3.3 incrementally reconstructs actions.

### Task T9.4.3.3 — Streaming action parser

`src/llm/action_parser.{hpp,cpp}` — incremental parser that
reads streamed tokens, emits `event::LlmAction{kind, content}`
as actions complete (close-paren).

The parser handles partial states: a `speak("hello wo` is
not yet a complete action; on `rld")` it emits
`Speak("hello world")`.

### Task T9.4.3.4 — Manager routing

`src/dialogue/manager.cpp`: subscribe to `LlmAction` instead
of (or alongside) the existing `LlmSentence` event. Action
handling:

- `Think(content)` → log at debug. Optionally publish
  `event::LlmThought` for M10 web UI to display.
- `Speak(content)` → split into sentences via existing
  `SentenceSplitter`, route to `TtsBridge` (M5 behavior for
  this content only).
- `Wait()` → FSM transitions to `Listening` without firing
  a turn-completed event. The user can speak again or the
  loop times out via M2 keep-alive.
- `Ignore()` → mark the user's preceding turn as `discarded`
  in SQLite (don't summarize, don't keep in context).

### Task T9.4.3.5 — System prompt extension

Each personality's system prompt gets a small GBNF-aware
appendix:

```
You respond using actions: think("..."), wait(), speak("..."),
ignore(). Use think() for reasoning the user shouldn't hear.
Use wait() if the user wasn't done speaking. Use ignore() for
non-speech (coughs, mumbles). Otherwise speak().
```

Translate appropriately for ru personalities.

### Task T9.4.3.6 — Tests

`tests/test_action_parser.cpp` — table driven, ~20 cases:

- Single `speak("hello.")` → one Speak action, content "hello.".
- Chained `think("...") speak("...")` → two actions in order.
- Partial input → no action emitted yet.
- Escape sequences in content (`\"`, `\\`).
- Malformed input → parse error (don't crash).

`tests/test_manager_action_routing.cpp` — Wait causes no TTS;
Ignore marks turn discarded; Think doesn't reach
SentenceSplitter.

### Task T9.4.3.7 — Acceptance

Run a 30-min soak with grammar on. Measure:

- `voice_llm_actions_total{kind}` distribution.
- "Inappropriate speak" rate (a `wait` situation answered
  with a `speak`) drops vs M8C baseline. Specifically for
  the M8C-known-bad pattern (assistant says
  "How can I help you?" after a discarded utterance) — this
  should fire ~zero times.

## Reuse hook for M11

The grammar infrastructure (`grammar.hpp`,
`action_parser.hpp`) is the same machinery M11.P11.2 uses
for tool-call output. Document the seam: M11 extends the
grammar with a `tool_call(name, args)` action; the parser
emits `event::ToolCall` instead of `LlmAction::Tool`; the
Manager routes to the tool framework instead of the action
handlers above.
