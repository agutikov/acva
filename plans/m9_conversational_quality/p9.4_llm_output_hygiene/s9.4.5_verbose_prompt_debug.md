# S9.4.5 — `--verbose-prompt` style debug for outgoing LLM requests

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 1-2 days
**Status:** stage placeholder.
**Shape:** debug observability task.

## Context

`ideas.md` line 23: `--verbose-prompt llama.cpp option`. Today
the operator has no direct visibility into what acva actually
sends to llama-server — chat template applied, system prompt
wired in, memory chunks injected. When prompts misbehave, the
debug loop is "diff acva log against an llm.txt I write by
hand".

## Tasks

### T9.4.5.1 — `cfg.llm.debug.dump_prompt`

When true, every outgoing chat-completions request body is
dumped (full messages array, grammar, sampling params) to a
debug log channel separate from the main JSON log stream
(too noisy for production).

### T9.4.5.2 — Per-turn dump file

Optional `cfg.llm.debug.dump_dir` — when set, each turn writes
`<turn_id>.request.json` and `<turn_id>.response.txt`. Easier
for operators to grep through after a session.

### T9.4.5.3 — `acva demo prompt-debug`

Self-contained demo: takes a single prompt, fires it through
the full PromptBuilder + memory injection, dumps the request,
and prints the streamed response. Useful for chat-template
audit work in S9.4.2.
