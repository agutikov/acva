# S9.4.1 — Remove filler / "How can I help you?" lead-ins

**Phase:** P9.4 (LLM output hygiene)  
**Estimate:** 2-3 days

## Context

Stock chat-tuned LLMs prepend boilerplate to nearly every reply:
"Of course!", "Great question!", "Let me help you with that.",
"How can I help you today?". For a voice assistant this is dead
weight — the user already knows they got a reply, the boilerplate
just adds 1-2 s of latency before the actual content.

Three levers, used together:

1. **Stop sequences** to abort if the model starts a
   filler-pattern.
2. **Logits bias** to reduce probability of common filler
   tokens.
3. **System-prompt pruning** — many filler patterns come from
   the personality's system prompt asking for "friendly tone";
   tighten the prompt.

## Acceptance

- Filler-rate on the soak prompt set drops ≥ 50% vs M8C
  baseline (measured by the existing soak-driver output: count
  of replies starting with one of ~20 known filler-prefixes).
- No regression in answer quality (manual review of 20
  random soak replies before / after).

## Tasks

### Task T9.4.1.1 — Filler corpus

Build `tests/fixtures/filler_prefixes.txt` — ~30 known filler
patterns:

```
Of course!
Sure!
Great question!
Let me help you
How can I help
I'd be happy to
Absolutely!
…
```

Include English + Russian (the supported langs).

### Task T9.4.1.2 — Stop sequences in `LlmClient`

`src/llm/client.cpp` `LlmRequest` already supports `stop`
arrays. Wire them from `cfg.llm.stop_sequences[]` and
`cfg.dialogue.filler_stop_enabled` (default true).

When the model emits a token that begins a filler prefix, the
stop matches and generation aborts. Combined with M7's
"cancel at next sentence boundary" the tts pipeline ignores
the aborted output.

### Task T9.4.1.3 — Logits bias

llama-server supports `logit_bias` in `/v1/chat/completions`.
Bias common filler tokens (the leading IDs of filler
patterns: "Of", "Sure", "Great", …) by `-3.0` (mild
discouragement, not banning).

Add `cfg.llm.filler_logit_bias` (map<string,float>) and
plumb through `LlmClient`. The map ships pre-populated for
the dialog default model; operators can extend per
personality.

### Task T9.4.1.4 — System-prompt audit

Read `config/default.yaml` `personalities.*.system_prompts.*`
for each shipped personality. Identify language asking for
"friendly", "helpful tone" — soften to "direct, concise"
where it shows up. The geek_enthusiast personality is fine
as-is (the filler is on-brand); consultant + bender should
be pruned.

### Task T9.4.1.5 — Measurement

Modify `scripts/soak-driver.py` to count filler-prefix hits
in replies: every assistant turn's `text` is matched against
the corpus, count goes into `soak-report.txt`. Run a 30-min
soak before + after the changes; compare.
