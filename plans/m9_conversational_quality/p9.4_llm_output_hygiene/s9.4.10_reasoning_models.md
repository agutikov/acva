# S9.4.10 — Reasoning model support

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 5-7 days
**Status:** stage placeholder.
**Shape:** investigation + impl, opens question Q16.

## Context

`ideas.md` line 123: "reasoning".

Reasoning models (DeepSeek-R1, QwQ, o1-style) emit a
`<think>...</think>` block before their natural-language
answer. Routed naively, the orchestrator would try to TTS
the entire reasoning trace — long, dry, unwanted aloud.

This stage decides how acva handles this output shape and
whether reasoning models become a first-class option in
the registry.

## Tasks

### T9.4.10.1 — Survey reasoning models that fit the budget

Catalog in `tools/acva-models` — which reasoning models
fit the 8 GB RTX 4060 budget? Likely: DeepSeek-R1-Distill-
Qwen-7B-Q4, QwQ-32B-Q3 (tight), maybe smaller distills.

### T9.4.10.2 — Reasoning-channel routing (Q16)

Three options:
- **A)** Pre-strip `<think>...</think>` at the LLM client
  layer. Cleanest separation; reasoning trace lost.
- **B)** Route the think block through `event::LlmThought`
  (S9.4.8 reuse). Surfaces in Web UI; not spoken.
- **C)** Treat it as `think("...")` action automatically —
  fold into the GBNF action grammar even when grammar is
  off.

### T9.4.10.3 — Implementation

Whichever option Q16 picks. If B: action parser learns to
recognise `<think>` boundary tokens and emit
`event::LlmThought`. If A: simple regex strip in
`LlmClient::process_chunk`.

### T9.4.10.4 — Acceptance

Drive a reasoning model on the soak prompt set. Spoken
output contains zero `<think>` content. UI/log surfaces
the thinking trace per Q16.
