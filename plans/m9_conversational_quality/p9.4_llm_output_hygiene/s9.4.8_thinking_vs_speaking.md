# S9.4.8 — Split internal LLM thinking from spoken output

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 5-7 days
**Status:** stage placeholder.
**Shape:** impl + opens question Q20.

## Context

`ideas.md` lines 39-40: "Can we split internal explicit LLM
thinking and speaking? Optionally enable verbal thinking."

S9.4.3 already lands a `think("...")` action that's logged
but not spoken. This stage extends it:

1. Make the thinking channel observable (Web UI, OTLP span,
   debug log) — not just dropped on the floor.
2. Add an *opt-in* "verbal thinking" mode where `think(...)`
   is spoken aloud at lower volume / different voice as
   commentary, then `speak(...)` is the real reply.

## Tasks

### T9.4.8.1 — `event::LlmThought`

New event carrying turn id + thought content. Emitted by
the action parser whenever a `think(...)` action completes.
Subscribed by: TurnWriter (persists separately from spoken
turns), M10 Web UI live-trace, OTLP span (M8B optional).

### T9.4.8.2 — Verbal-thinking mode (Q20)

`cfg.dialogue.verbal_thinking.{enabled, voice, volume_db,
prefix}` — when on, `think(...)` content is voiced through
the TTS pipeline at a quieter setting and a marker
("...") is prepended.

Q20 decides the routing shape: separate parallel TTS run vs
inline-as-prefix vs operator-only-debug-mode.

### T9.4.8.3 — Manager wiring

Manager subscribes to `LlmThought` in addition to existing
`LlmAction::Speak`. When verbal thinking is enabled,
thoughts go through TtsBridge with a tagged voice override;
otherwise they flow only to log + UI.

### T9.4.8.4 — Tests

Action parser emits one `LlmThought` and one `LlmSentence`
when input is `think("foo") speak("bar")`. With verbal
thinking off, only one TTS request. With verbal thinking
on, two TTS requests, second strictly after the first
finishes (or pipelined per Q20).

### T9.4.8.5 — UX validation

Operator dogfood: with verbal thinking on, does it feel
"the assistant is thinking aloud" (good) or "the assistant
won't shut up" (bad)? Capture in `docs/reports/`.

## Resolves / depends

- **Opens Q20** — verbal-thinking routing shape.
- Depends on S9.4.3 (action grammar) for the source signal.
- Forward-link: M10 Web UI dialogue trace surfaces
  `LlmThought` as a distinct lane.
