# S9.4.9 — Two-stage generation (draft + refine)

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 1 week
**Status:** stage placeholder.
**Shape:** investigation + opens question Q17.

## Context

`ideas.md` line 128: "2-stage generation".

A draft-then-refine pass can lift quality on multi-sentence
replies (the draft establishes structure, the refine pass
tightens phrasing). Cost: latency doubles; quality lift may
not survive A/B against simply running a larger model
once.

This stage runs the experiment, then either ships a config
toggle or files a "not-now" close-out.

## Tasks

### T9.4.9.1 — Experiment harness

Drive 50 prompts through three configs:
- `single` — current single-pass.
- `two_stage_same_model` — draft + refine on the same
  llama.
- `two_stage_two_model` — small fast draft + larger
  verifier (requires multi-model from M12.P12.5; gate
  on that landing).

Capture mean tokens, P95 latency, and 1-5 manual quality
score per response.

### T9.4.9.2 — Resolve Q17

Three options:
- **A)** Skip — single-pass remains the only mode.
- **B)** Same-model two-stage as opt-in (`cfg.llm.refine.
  enabled`).
- **C)** Two-model only — gated on M12.P12.5 landing
  multi-model.

Decision lands in `m9_conversational_quality/
open_questions.md` Q17.

### T9.4.9.3 — Ship the chosen path

If B: add the refine pass + test that cancellation cascades
through both stages. If C: open a follow-on task in M12.P12.5.
If A: close the stage with the experiment data.
