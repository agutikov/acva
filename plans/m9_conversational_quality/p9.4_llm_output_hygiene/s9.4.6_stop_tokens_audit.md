# S9.4.6 — Stop-token audit + tokenizer control tokens

**Phase:** P9.4 (LLM output hygiene)
**Estimate:** 2-3 days
**Status:** stage placeholder.
**Shape:** investigation + config exposure.

## Context

`ideas.md` line 22: "stop token", and lines 31-34: "Clarify
location and configuration of: chat template / special
tokenizer control tokens / GBNF and output formatting".

llama.cpp picks up stop tokens from the GGUF metadata; some
fine-tunes ship wrong values, and chat-template-baked stop
tokens are sometimes silently ignored under `chat-template`
mode. We have no test or audit for this today.

## Tasks

### T9.4.6.1 — Audit script

`scripts/llm-stop-token-audit.py` — given a model alias,
queries `/props` and `/v1/models` on llama-server, prints
detected EOS / EOT / stop tokens, the chat template, and any
override that `cfg.llm` config applied.

### T9.4.6.2 — Config override surface

`cfg.llm.stop_tokens: [string]` — explicit list that gets
appended on the request. Operator escape hatch when the
GGUF metadata is wrong.

### T9.4.6.3 — Document the tokenizer surface

`docs/guide/llm-internals.md` (new) — one page covering:
where the chat template lives (GGUF metadata vs server
runtime override), how to override it, special tokens
catalog per model family we ship, and the stop-token
override path above.
