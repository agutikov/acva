# S9.1.2 — `PartialTranscript` event publication

**Phase:** P9.1 (streaming partials source)  
**Estimate:** 1-2 weeks  
**Depends on:** S9.1.1 strategy resolved.

## Context

Once the chosen STT path emits deltas, the M5 dispatcher
already routes
`conversation.item.input_audio_transcription.delta` →
`PartialTranscript` event. What's missing today and lands
here:

- The event type itself (struct + bus subscription glue —
  partials currently arrive but no one listens).
- `stable_prefix_len` field — distance into `text` that
  hasn't changed across the last N partials. The dispatcher
  computes via longest-common-prefix against the previous
  emission per turn.
- `seq` — monotonic counter per turn (already wired in M5
  for the buffered final; reuse).

## Acceptance

- `event::PartialTranscript` lands with fields:
  `{turn_id, text, stable_prefix_len, seq, captured_at}`.
- `RealtimeSttClient::on_partial` callback publishes onto the
  bus (currently dispatches into a no-op).
- Two unit tests: stable-prefix tracking across a sequence
  of partials (text grows, prefix grows monotonically); seq
  monotonic per turn, resets across turns.
- One integration test: live STT, run a 5 s recorded fixture,
  observe ≥ 3 distinct `PartialTranscript` events before the
  `FinalTranscript`.

## Tasks

### Task T9.1.2.1 — Event type + bus wiring

Add `event::PartialTranscript` to `src/event/event.hpp`. Wire
through to `EventBus::subscribe` like the existing event
families. Update `src/orchestrator/observability/event_tracer.cpp`
to log partials at debug level (so `--log-level=debug`
operators can see them stream).

### Task T9.1.2.2 — Stable-prefix tracking in dispatcher

`src/stt/realtime_event_dispatch.cpp` already accumulates the
delta text. Add a `last_text_per_turn` map; on each new
partial, compute the longest common prefix vs the prior
text, set `stable_prefix_len = LCP.size()`. Reset on
`SpeechStarted` (new turn).

Edge case: deltas can shrink (Whisper sometimes revises
mid-utterance — "hey" → "hi"). When `text` becomes shorter
than the prior `text`, the LCP is up to the shorter length;
`stable_prefix_len` may decrease. Document this in the event
struct comment so consumers don't assume monotonic growth.

### Task T9.1.2.3 — Real STT path: emit partials

Two sub-tasks gated on S9.1.1 outcome:

**If S9.1.1 = A (Speaches PR):**
- Land the Speaches PR (or wait on review).
- `RealtimeSttClient::on_partial` already gets called from
  the realtime data channel; just publish onto the bus
  instead of swallowing.

**If S9.1.1 = B (side-car):**
- Add the side-car HTTP client (cpp-httplib SSE consumer)
  as a new subsystem under `src/stt/`.
- Wire it parallel to `RealtimeSttClient` — same
  `audio::CaptureFrame` subscription, separate output via
  `event::PartialTranscript`.
- The buffered `FinalTranscript` still comes from Speaches
  through the existing realtime path.

### Task T9.1.2.4 — Tests

`tests/test_partial_transcript_dispatch.cpp` (new) — table
driven. Inputs: sequence of (delta text, expected stable
prefix len). Exercise growth, shrink, and turn boundary.

`tests/test_realtime_partials_smoke.cpp` (integration) — run
a known-good 5 s audio fixture (e.g. the existing
`scripts/wake-word-offline.sh` output) through the realtime
client, assert ≥ 3 partials with monotonic seq.
