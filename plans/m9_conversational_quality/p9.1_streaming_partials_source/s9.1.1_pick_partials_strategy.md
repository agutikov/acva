# S9.1.1 — Pick the partial-transcript strategy

**Phase:** P9.1 (streaming partials source)  
**Estimate:** 1 week (time-boxed)  
**Open question:** decided here, recorded in [`m9_conversational_quality/open_questions.md`](../open_questions.md) as the next free `Q<n>` ID (continuing global numbering — Q15 if no other milestone added one first).

## Context

Speaches' realtime endpoint as of 2026-05-02 doesn't emit
`conversation.item.input_audio_transcription.delta` — see
[`docs/history/MVP/open_questions.md`](../../../docs/history/MVP/open_questions.md)
§L6. P9.1 needs partial transcripts to feed P9.2 speculation
and P9.3 adaptive endpointer. Three options on the table; pick
one in the first week of the phase, fall back to Option B if
the chosen path stalls.

## Options

### Option A — PR Speaches to emit `transcription.delta`

faster-whisper supports streaming via the generator API.
Speaches' `realtime/input_audio_buffer.py` `InputAudioBufferTranscriber._handler`
would iterate segments and publish
`ConversationItemInputAudioTranscriptionDeltaEvent` per word,
then `…CompletedEvent` at end. ~50 LoC PR upstream.

**Pros:** smallest in-tree change. Reuses existing M5 wiring.  
**Cons:** depends on upstream merge timing. Speaches review
cadence is unpredictable.

### Option B — Side-car streaming Whisper

Add a second backend container (e.g., `whisper.cpp/examples/stream`
wrapped by a thin HTTP+SSE server) alongside Speaches. STT
forks: the realtime data channel still drives buffered finals
through Speaches, but a parallel SSE consumer feeds partials
from the side-car.

**Pros:** no upstream dependency. Fail-safe — if Speaches
recovers partials we drop the side-car.  
**Cons:** ~1.5 GB extra VRAM (Whisper small / medium). Two
audio paths increases complexity. Extra Compose service to
maintain.

### Option C — Re-platform STT entirely

Drop Speaches for STT, keep for TTS. Move STT to a custom
whisper.cpp wrapper that streams natively.

**Pros:** maximum flexibility.  
**Cons:** multi-week scope. Most of M5 runs cleanly through
Speaches today; throwing it away is wasteful.

## Acceptance

- A choice is made and written into
  [`m9_conversational_quality/open_questions.md`](../open_questions.md)
  (one of A/B/C), with the rationale + the consequence.
- If A: a Speaches PR is open with the linked branch and a
  realistic ETA. Side-car fallback (B) plan documented in
  case PR stalls > 2 weeks.
- If B: Compose service is wired, VRAM budget calculated and
  fits 8 GB workstation alongside llama + Speaches.
- If C: M5's STT client is decoupled — `RealtimeSttClient`
  stays as the M5 fallback, new client owns the streaming
  path.

## Tasks

### Task T9.1.1.1 — Speaches reconnaissance (1 day)

Re-read `realtime/input_audio_buffer.py` against the latest
Speaches release. Has anything changed since 2026-05-02?
Check upstream Speaches issue tracker for any mention of
`transcription.delta`. Compare against
[`project_m5_realtime_spike.md`](../../../docs/history/MVP/) memory note
to know what we already verified.

### Task T9.1.1.2 — VRAM budget for Option B (1 day)

If Option A looks slow / dead, run Option B numbers:

- Whisper small (244 MB) vs. medium (769 MB) vs. large (~3 GB)
  for the side-car partials source.
- VRAM with llama 7B Q4 (~5 GB) + faster-whisper turbo
  (~1.6 GB) + Piper voices (~250 MB) baseline ~6.8 GB.
- Headroom on 8 GB 4060: ~1.2 GB. Whisper small fits; medium
  is tight; large doesn't.

Pick the smallest model that gives usable partial quality.

### Task T9.1.1.3 — Decision + write-up (3 days)

Choose A or B (C only if both fail decisively). Write the
chosen approach as a new resolved entry in
[`m9_conversational_quality/open_questions.md`](../open_questions.md)
under the next free `Q<n>` ID (Q15 if no other milestone has
allocated one yet):

```markdown
## Q15. Partial-transcript source — *(resolved YYYY-MM-DD)*
- Decision: <A | B>
- Reason: <one paragraph>
- Consequence: <PR URL | Compose service path>
- Fallback: <if chosen path fails by date X, switch to Y>
```

If the answer is B, S9.1.2 picks up the side-car wiring; if A,
S9.1.2 waits on the PR and instead writes the consumer code
(harmless once Speaches starts emitting deltas).
