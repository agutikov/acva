# S9.3.2 — Address detector — heuristic baseline

**Phase:** P9.3 (adaptive endpointer + address detection)  
**Estimate:** 3-4 days  
**Depends on:** S9.3.1 (re-uses partial-transcript subscription).

## Context

The conversational-mode product line wants to know "is this
addressed to me?" before turning every detected utterance into a
turn. Wake-word answers a different question (did the user say
the magic phrase). Address detection is the right primitive for
multi-occupant rooms.

Two-stage approach across S9.3.2 + S9.3.3:

- **S9.3.2 (this stage)** — ship a heuristic baseline that's
  predictable, debuggable, and good enough for solo use.
- **S9.3.3** — replace or augment with a learned classifier
  once we have production traces to train on.

The heuristic doesn't try to be smart. It says "address detected
if any of: utterance starts with assistant name; OR very short
+ ends in `?`; OR partial contains a strong imperative head."
That suppresses talk between people *to each other* (long form,
non-imperative, no name) while keeping the addressed-to-me
case responsive.

## Tasks

### Task T9.3.2.1 — `dialogue::AddressDetector` interface

`src/dialogue/address_detector.{hpp,cpp}` — pure logic,
subscribes to `PartialTranscript` and `FinalTranscript`,
publishes `event::AddressDecision{turn_id, addressed: bool,
confidence: float, cause: string}`.

```cpp
class AddressDetector {
public:
    explicit AddressDetector(const AddressConfig& cfg);
    AddressDecision evaluate(const event::PartialTranscript& p);
    AddressDecision evaluate(const event::FinalTranscript& f);
};

struct AddressConfig {
    bool enabled = true;
    std::vector<std::string> assistant_names;     // ["acva", "jarvis", …]
    int  max_short_query_words = 8;
    std::vector<std::string> imperative_heads;    // ["turn", "set", "play", …]
};
```

### Task T9.3.2.2 — Heuristic rules

Decision is `addressed = true` if any of:

- **Name-prefix:** first word matches one of
  `cfg.assistant_names` (case-insensitive). cause=`name_prefix`.
- **Short-question:** ≤ `max_short_query_words` AND ends in `?`.
  cause=`short_question`.
- **Imperative head:** first content word is in
  `cfg.imperative_heads` ("turn off the lights",
  "set an alarm"). cause=`imperative`.

Otherwise `addressed = false`, cause=`no_signal`.

`confidence` is a flat 0.7 for any positive case (the
heuristic doesn't have a real probability; the classifier in
S9.3.3 will).

### Task T9.3.2.3 — Manager integration

`src/dialogue/manager.cpp`: subscribe to `AddressDecision`.
On `FinalTranscript`, look up the most recent decision for
that turn. If `addressed = false`, drop the turn (mark as
`discarded` in SQLite, no LLM call). If `addressed = true`,
proceed normally.

When wake-word is also enabled (`cfg.audio.wake_word.enabled`),
address-detection is **AND** with the wake gate — both must
agree. Off-by-default operators get pure address detection.

### Task T9.3.2.4 — Validation fixtures

Build a small test corpus under `tests/fixtures/address/`:

- `addressed/*.wav` — 20 utterances clearly addressed
  ("hey acva, what time is it?", "set an alarm",
  "turn off the lights", …).
- `not_addressed/*.wav` — 20 utterances of room talk
  ("did you see the game last night", "I think we should",
  "tomorrow we have to", …).

Generate via Speaches TTS over multiple voices to avoid
single-speaker overfitting.

`tests/test_address_detector_heuristic.cpp` runs the corpus
through STT + the heuristic, asserts:

- Addressed → addressed ≥ 90% (sensitivity).
- Not-addressed → not-addressed ≥ 80% (specificity).

If specificity is below 80%, log the failures — they become
the training set for S9.3.3.

### Task T9.3.2.5 — Metrics

`voice_address_decisions_total{outcome="addressed|not_addressed",cause}`
counter.
`voice_address_dropped_total{cause}` counter for the
"would have been a turn but address gate dropped it" path.
