# S9.3.1 — Adaptive endpointer

**Phase:** P9.3 (adaptive endpointer + address detection)  
**Estimate:** 4-5 days  
**Depends on:** P9.1 partials.

## Context

Today: `Endpointer::on_frame` fires `SpeechEnded` after a fixed
`cfg.vad.hangover_ms` (default 600 ms) of below-offset audio.
Wrong at both ends:

- Mid-thought pause ("set an alarm for… uh… seven AM") —
  600 ms is too short, user gets cut.
- Crisp finish ("turn off the lights.") — 600 ms is too long,
  300 ms of silence is plenty.

Adaptive endpointer reads the **partial transcript itself** —
hesitation markers ("um", "uh", "…"), trailing conjunctions
("and", "or", "but") signal "more coming"; terminal
punctuation, full sentence shape signals "I'm done".

## Tasks

### Task T9.3.1.1 — `audio::AdaptiveEndpointer`

`src/audio/adaptive_endpointer.{hpp,cpp}` — wraps the existing
`Endpointer`, augments its decision with partial-transcript
signal.

```cpp
class AdaptiveEndpointer {
public:
    AdaptiveEndpointer(const Endpointer::Config& base,
                       const AdaptiveConfig& adaptive);
    void on_frame(const Frame& f);
    void on_partial(const event::PartialTranscript& p);
    // existing Endpointer surface: SpeechStarted/Ended events
};

struct AdaptiveConfig {
    bool   enabled              = true;
    int    short_hangover_ms    = 250;   // for crisp finishes
    int    long_hangover_ms     = 1200;  // for hesitation
    float  hesitation_marker_weight = 0.7;  // bias toward long
    float  terminal_punct_weight    = 0.7;  // bias toward short
};
```

### Task T9.3.1.2 — Signal extraction

Two heuristics on the partial text (run when partial arrives):

- **Terminal-shape** — text ends with `.`, `!`, `?` or matches
  a full-sentence regex (`/^[A-Z][^.!?]+[.!?]$/`).
- **Hesitation-shape** — last token is in
  `{um, uh, er, hmm, well, …}` or in `{and, or, but, so, …}`,
  or partial ends with an ellipsis.

Mix into a single `decision_score ∈ [0, 1]`:

```
0.0 = strong hesitation → use long_hangover_ms
1.0 = strong terminal   → use short_hangover_ms
0.5 = neutral           → use base hangover (M5 default)
```

Apply during `on_frame`: when below-offset, use
`hangover_ms_for(decision_score)` instead of the fixed value.

### Task T9.3.1.3 — Multilingual support

Hesitation markers + sentence-shape regex are language-aware.
Read the detected language from the most recent partial (M5
already routes language). Ship en + ru lists initially; add
others as fixtures land.

`config/default.yaml` gets a new `vad.hesitation_markers` map
keyed by lang code, and `vad.terminal_pattern` regex map.
