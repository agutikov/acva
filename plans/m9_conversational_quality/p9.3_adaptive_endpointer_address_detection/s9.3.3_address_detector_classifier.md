# S9.3.3 — Address detector — learned classifier

**Phase:** P9.3 (adaptive endpointer + address detection)  
**Estimate:** 5-7 days (most likely punted to a P9.3 v2 round)  
**Depends on:** S9.3.2 + production traces.

## Context

The S9.3.2 heuristic is deliberately dumb-but-working. Once
acva has been running with it for a while and we have logs of
addressed-vs-not utterances, replace or augment with a small
text classifier.

This stage is **time-boxed at 1 week**. If the heuristic's
specificity from S9.3.2 lands ≥ 85% on the validation corpus,
**skip this stage** — diminishing returns aren't worth the
training infra.

## Tasks (only if heuristic specificity < 85%)

### Task T9.3.3.1 — Training corpus

Collect:

- `tests/fixtures/address/` from S9.3.2 (40 labelled).
- Production traces: extract `FinalTranscript` from
  `acva.db` over the dogfood window, manually label
  ~200 utterances. Privacy: stays on dev workstation; no
  upload.
- Synthetic augmentation: Speaches TTS of GPT-generated
  addressed-vs-room-talk pairs (~500). Document the
  generation prompt for reproducibility.

### Task T9.3.3.2 — Model choice

Tiny text classifier — fits in CPU inference budget (no
extra VRAM). Options:

- **Distilled BERT** — ONNX export, ~25 MB.
- **fastText** — ~5 MB, lightning-fast, good enough for
  binary classification.
- **Heuristic + logistic regression on word features** —
  smallest, completely interpretable.

**Recommendation:** start with logistic regression. If
accuracy plateaus < 90%, escalate to fastText.

### Task T9.3.3.3 — `tools/train-address-detector`

Mirror `tools/train-wake-word`'s shape: Python 3 driver,
opt-in `pip install` deps, registers the trained model under
`${XDG_DATA_HOME}/acva/models/address/<name>.onnx` (or
`.bin` for fastText).

### Task T9.3.3.4 — Wire classifier into AddressDetector

`audio::AddressDetector` runs the heuristic + the classifier
in parallel; final decision is the OR of "either says
addressed" with confidence = max. The classifier path is
gated on a model file existing under
`${XDG_DATA_HOME}/acva/models/address/`; without one, only
the heuristic runs.

Avoids the all-or-nothing failure: if the classifier is
worse than the heuristic on some category, the OR keeps
sensitivity.

### Task T9.3.3.5 — Validation

Re-run the S9.3.2 corpus + a held-out 50-sample test set.
Acceptance: specificity ≥ 90% with sensitivity ≥ 90%.
Document the trade-off curve in
`docs/guide/operations.md`.

## Deferred / out-of-scope

- Acoustic features (intonation, speech rhythm) — text-only
  for v1. M12 voice biometric ID may add a "user is the
  primary user" gate that helps, but it's a different
  signal.
- Per-user address style — stays generic in M9; per-user
  becomes M12.P12.2 territory.
