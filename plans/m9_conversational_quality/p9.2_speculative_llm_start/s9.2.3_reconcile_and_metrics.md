# S9.2.3 — Metrics, soak, acceptance

**Phase:** P9.2 (speculative LLM start)  
**Estimate:** 3-4 days  
**Depends on:** S9.2.2 reconcile working.

## Context

Close the phase: ship metrics so we can measure whether
speculation is paying off, run a short soak with speculation
enabled, document the latency win.

## Acceptance

- Metrics shipped: `voice_speculation_kept_total`,
  `voice_speculation_cancelled_total{cause="diverged|barge_in|timeout"}`,
  `voice_speculation_first_token_ready_ms` histogram,
  `voice_speculation_promote_savings_ms` histogram.
- Grafana panel showing speculation hit rate + latency win.
- Soak run: 30 min with speculation on, hit rate ≥ 30%
  (kept / (kept + cancelled)) on the soak prompt set.
- Documented latency win in `docs/guide/operations.md`
  § Performance: P50 first-token-ready drops by ≥ 200 ms on
  speculation-eligible utterances.

## Tasks

### Task T9.2.3.1 — Metrics registration

`src/metrics/registry.cpp`: register the four new series.
Wire from Manager:

- On `GateDecision::Start`: increment a "speculation
  started" counter.
- On promote: increment kept; record
  `promote_savings_ms` (= time saved vs starting LLM at
  FinalTranscript).
- On cancel: increment cancelled with cause label.
- On every speculative LLM `LlmFirstToken`: record
  `first_token_ready_ms` from `SpeechStarted`.

### Task T9.2.3.2 — Grafana panel

Add a "Speculation" panel to `packaging/grafana/acva.json`:

- Hit rate gauge: `kept / (kept + cancelled)` over 5 min.
- Latency histogram heatmap:
  `voice_speculation_promote_savings_ms`.
- Cancel-cause breakdown bar chart by `cause` label.

### Task T9.2.3.3 — Soak run + measurement

Run `scripts/soak.sh 30m` with `cfg.dialogue.speculation.enabled:
true`. Verify:

- Hit rate ≥ 30%.
- No new error patterns in `voice_stuck_total` (the watchdog
  should not fire more often than baseline).
- RSS growth in budget (same gate as M8B soak).

Compare end-of-soak metrics against M8B baseline; document
delta in `docs/guide/operations.md` § Performance.

### Task T9.2.3.4 — Acceptance write-up

Update `plans/m9_conversational_quality/README.md` § Acceptance
gate 2 with the actual measured savings number. Mark P9.2
closed in the milestone README phase table.
