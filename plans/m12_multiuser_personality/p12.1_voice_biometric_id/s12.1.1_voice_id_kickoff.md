# S12.1.1 — Voice biometric ID kickoff

**Phase:** P12.1 (voice biometric ID)
**Estimate:** 3-4 weeks
**Status:** stage placeholder.

## Likely tasks

### T12.1.1.1 — Model choice

ECAPA-TDNN vs pyannote vs lightweight ResNet variants.
Constraints: CPU-feasible (no extra VRAM beyond M5 budget),
ONNX-exportable, ~10 MB on disk. Speaker-embedding output
~192 dims is the de-facto standard.

### T12.1.1.2 — Enrollment flow

`acva users enroll <name>` — operator records 3 × 10 s
clips of normal speech; embeddings averaged into a per-user
centroid. Persists to `~/.local/share/acva/users/<name>.bin`
(centroid + metadata).

### T12.1.1.3 — Identification at utterance time

On each `UtteranceReady`, compute the embedding, compare
cosine vs every enrolled centroid. Threshold-gate: above
0.65 → identified, otherwise → "unknown". Fast enough to
not add measurable latency to the dialogue path (<10 ms on
the dev box).

### T12.1.1.4 — `event::UserIdentified`

New event with `user_id` (string) or "unknown" + confidence.
Fired between `FinalTranscript` and the LLM start so
PromptBuilder can include "user is X" in context.

### T12.1.1.5 — Tests

Synthetic: TTS multiple voices, enroll subset, verify
identification accuracy on held-out clips.
