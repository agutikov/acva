# S12.5.1 — Multi-context with one llama-server

**Phase:** P12.5 (multi-context / multi-model runtime)
**Estimate:** 1 week
**Status:** stage placeholder.
**Shape:** investigation.

## Context

`ideas.md` line 103: "multi-context with single model".

llama.cpp's `llama-server` supports parallel slots
(`-np N` flag), each holding an independent KV cache for
a separate conversation. Concurrent personalities, parallel
NPC characters, or per-user contexts could all exploit
this without loading a second model.

## Tasks

### T12.5.1.1 — Capacity survey

For the budget LLM (llama-7B-Q4 on RTX 4060):
- max parallel slots before OOM
- per-slot context window (ctx-size / N)
- prompt-eval throughput per slot
- token-gen throughput per slot

Some of this is already documented in llama.cpp; verify on
acva's actual model.

### T12.5.1.2 — Orchestrator wiring sketch

`LlmClient` becomes a slot pool. Each request carries a
slot tag; M8A model-controller learns slot allocation.
Document the full code-path that would change.

### T12.5.1.3 — Switch cost

Switching a slot to a new system prompt is *free* if KV
cache is reset, but you lose context. Switching to an
*existing* slot's context is free if the slot is
preserved across turns.
