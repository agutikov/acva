# S12.5.3 — Persona-per-context wiring (gated)

**Phase:** P12.5 (multi-context / multi-model runtime)
**Estimate:** 1-2 weeks
**Status:** stage placeholder.
**Shape:** impl, gated on Q23.

## Context

If Q23 picks "ship multi-context with one model" (S12.5.1)
this is the implementation slot. If Q23 picks anything
else, this stage closes with a write-up.

## Tasks (when gated open)

### T12.5.3.1 — Slot-tagged LlmClient

`LlmClient::submit` accepts `SlotId`. Pool sized by
`cfg.llm.max_slots`. Slot lookup keyed by
`(user_id?, personality_name)`.

### T12.5.3.2 — DialogueManager extension

Manager owns the slot mapping; on speaker switch
(M12.P12.3) or personality switch (M12.P12.4 voice
command), select the right slot. Cancel-cascade isolates
to the active slot only.

### T12.5.3.3 — Eviction policy

LRU when active count exceeds `max_slots`. Persisted
context survives slot eviction (replay from DB on
re-activation if cheap; otherwise cold-start).

### T12.5.3.4 — Tests

Two parallel personalities, alternating turns, neither
sees the other's context. Cancel on personality A doesn't
disturb personality B.

## Resolves / depends

- **Gated on Q23.**
- Depends on M8A model-controller (slot count is part of
  the load request).
- Depends on M12.P12.4 personality switch path.
