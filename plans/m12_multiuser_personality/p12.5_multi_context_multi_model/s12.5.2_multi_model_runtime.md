# S12.5.2 — Multi-model runtime

**Phase:** P12.5 (multi-context / multi-model runtime)
**Estimate:** 1 week
**Status:** stage placeholder.
**Shape:** investigation.

## Context

`ideas.md` line 104: "multi-model runtime".

Different LLMs for different roles at the same time:
- a small fast model for filler / address detection
  classifier
- a large model for the user-facing reply
- a different model for a specialist personality

Constraint: 8 GB VRAM. Two simultaneously-resident large
models is impossible; small + medium is tight; small +
small + Speaches is feasible.

## Tasks

For each candidate pair (e.g. llama-3B-Q4 + llama-7B-Q4),
measure idle + concurrent inference VRAM. Account for
Speaches (~2 GB) and STT pre/post buffers.

Two ports / two `llama-server` processes? One process

refactor. Document the cost.

(personality binds to model)? Dynamic (small model for
short turns, large for long)? Speculative-decoding pair?
