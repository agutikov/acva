# S12.5.4 — GPU budget under concurrency

**Phase:** P12.5 (multi-context / multi-model runtime)
**Estimate:** 4-5 days
**Status:** stage placeholder.
**Shape:** investigation.

## Context

The `project_gpu_cdi_and_vram.md` memory note says the 8 GB
RTX 4060 already runs llama-7B-Q4 + faster-whisper-large-v3-
turbo within budget. Concurrency on top of that needs careful
accounting.

## Tasks

### T12.5.4.1 — Inventory

Every GPU consumer in acva:
- llama-server (~4.5 GB at runtime)
- Speaches STT (faster-whisper-large-v3-turbo, ~1.2 GB
  pinned via `WHISPER__TTL=-1`)
- Speaches TTS (variable; depends on backend)
- Optional RVC (S9.5.3, ~500 MB-1 GB)
- Optional second llama (S12.5.2)

Document worst-case concurrent peaks.

### T12.5.4.2 — Wedge probability

The M8B speaches-wedge classifier is empirical evidence
that concurrent VRAM contention does cause failure. Run a
soak with concurrent contexts (S12.5.3 prototype) and
measure wedge rate vs single-context baseline.

### T12.5.4.3 — Operator guidance

Output: a `docs/guide/gpu-budget.md` that says "if you
enable RVC you must lower llama context to N tokens" etc.
Concrete configurations that work and fail.

### T12.5.4.4 — Q23 input

Combined with S12.5.1 + S12.5.2, this is the third leg of
the Q23 decision data set.

## Resolves / depends
