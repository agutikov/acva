# S12.2.1 — Per-user memory + attribution

**Phase:** P12.2 (per-user memory + attribution)
**Estimate:** 2-3 weeks
**Status:** stage placeholder.

## Likely tasks

### T12.2.1.1 — Schema migration

Add `user_id` column to `sessions`, `turns`, `facts`. NULL
allowed for the unknown / pre-M12 case. Migration on
startup.

### T12.2.1.2 — Memory query scoping

`MemoryThread::read` queries gain a per-user filter. Default
behavior: include the active speaker's facts AND public
facts (NULL user_id). Cross-user facts (one user mentioning
another) get an explicit user_id flag.

### T12.2.1.3 — "Remember main user by voice" UX

First enrolled user becomes the default. Unidentified
utterances assume the main user if confidence margin is
small. Operator can override via
`acva users set-main <name>`.

### T12.2.1.4 — Privacy boundary

`POST /wipe?user=<name>` — delete all sessions / turns /
facts attributed to one user. Cascades like the existing
session wipe.

### T12.2.1.5 — Web UI surfacing (M10 reuse)

Sessions list filters by user; per-user memory view in the
UI from M10.P10.4.
