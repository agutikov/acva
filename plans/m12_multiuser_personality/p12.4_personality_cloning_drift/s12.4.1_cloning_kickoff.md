# S12.4.1 — Personality cloning + drift policy

**Phase:** P12.4 (personality cloning + drift)
**Estimate:** 2-3 weeks
**Status:** stage placeholder.
**Open question:** Q8 (drift policy) — decided here.

## Likely tasks

### T12.4.1.1 — Style fingerprint

Extract per-user / per-personality style features from
session traces: vocabulary distribution, sentence length,
formality, hedge frequency. Persists as a vector +
exemplars.

### T12.4.1.2 — Clone seeding

`acva personality clone <source> <new>` — copies the
source personality's system prompt, swaps in the
fingerprint as additional context, registers under
`personalities.<new>:`.

### T12.4.1.3 — Drift policy (Q8)

How much can the clone drift before it stops being
recognizable? Cosine similarity to the seed fingerprint
must stay ≥ 0.8 (TBD threshold). Re-fingerprint
periodically; if drift exceeds bound, alert the operator
or auto-revert.

### T12.4.1.4 — Switch personality command

Voice command "switch to <name>" → reload personality
overlay live (M8A reload-restart story; personality is
restart-required, so this triggers a warm restart with
the new active personality).

### T12.4.1.5 — Validation

Operator clones their consultant personality, runs 30
turns, manually rates clone-vs-original similarity.
Re-runs with drift-bound enforced; verifies clone stays
on style.
