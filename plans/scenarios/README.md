# Niche scenarios

Investigation parking lot for niche use cases that don't fit
the M9-M15 roadmap but shouldn't be lost. Each file is one
scenario with its own status + trigger condition.

These are deliberately *not* milestones. They become
milestones only if a real user materialises with the use
case. Until then, they live here as ready-to-pick-up
sketches.

`plans/open_questions.md` Q4 is the parent; each file below
is one expansion of Q4.

## Scenarios

| File | Trigger condition |
|---|---|
| [`game_npc_character_design.md`](game_npc_character_design.md) | Real game-design dogfood request |
| [`voice_translation.md`](voice_translation.md) | Real translation user with a benchmark |
| [`tg_on_call_assistant.md`](tg_on_call_assistant.md) | Real on-call rotation that wants this |

## Conventions

- Status: `parked` (default), `pre-research`, `picked-up
  → m<n>.<phase>` if promoted.
- Each file has: scenario, why it's interesting, why it's
  parked, what would change to pick it up, and the
  rough shape of an investigation if/when it does start.
- Promote to a milestone phase by moving the content into
  the right milestone folder. Leave a one-line stub here
  pointing at the new home.
